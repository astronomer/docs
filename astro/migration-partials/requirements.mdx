To complete this guide, you will need at a minimum:
- Access to [Astro Cloud](https://cloud.astronomer.io) and the [ability to create workspaces](manage-workspaces.md)
- The `astro` CLI [installed](cli/install-cli.md) and [configured](log-in-to-astro.md#log-in-to-the-astro-cli) - *note that this requires a functional `docker` installation*
- Read access to the source Airflow environment to migrate
- Read access to the cloud storage bucket utilized by the source environment for DAG code, *if one exists*
- Read access to any source control supporting your source Airflow environment, *(e.g. Github)*
- Create access for new source control repositories for your new Astro Airflow
- Access to Secrets Backend credentials, *if in use in the source Airflow*
- Administrative access to create new CI/CD pipelines, *if in use in the source Airflow*
